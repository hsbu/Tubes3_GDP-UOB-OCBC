Test 4: Skip Tags & Edge Cases

Menguji apakah ekstensi dengan benar mengabaikan keyword di dalam elemen yang seharusnya dilewati, dan tetap mendeteksi keyword di teks yang terlihat pengguna.

1. Keyword di dalam <script> — HARUS diabaikan

Ekspektasi: TIDAK terdeteksi

Tidak ada keyword di atas yang seharusnya dideteksi (ada di dalam script tag).

2. Keyword di dalam <style> inline — HARUS diabaikan

Ekspektasi: TIDAK terdeteksi

CSS di atas tidak seharusnya dideteksi sebagai keyword.

3. Keyword di dalam <code> dan <pre> — HARUS diabaikan

Ekspektasi: TIDAK terdeteksi
    // Contoh kode
    if (user.plays("slot") || user.wins("jackpot")) {
      grantBonus("freespin");
    }

Keyword: gacor, maxwin, scatter di dalam code tag tidak boleh dideteksi.

4. Keyword di dalam <textarea> — HARUS diabaikan

Ekspektasi: TIDAK terdeteksi

5. Keyword di atribut HTML — HARUS diabaikan

Ekspektasi: TIDAK terdeteksi (atribut bukan text node)
Link biasa

6. Keyword di teks terlihat — HARUS terdeteksi

Ekspektasi: HARUS terdeteksi

Di bawah ini adalah konten yang benar-benar terlihat pengguna dan berisi keyword:

Mainkan slot dan menangkan jackpot besar!

Fitur scatter dan wild tersedia di semua permainan pragmatic.

Dapatkan bonus dan freespin setiap hari dari situs gacor.

7. Teks sangat pendek (< 2 karakter) — HARUS diabaikan

Ekspektasi: TIDAK terdeteksi (terlalu pendek untuk diproses)

SLOT

Kata di atas dipisah per huruf, masing-masing node hanya 1 karakter.

8. Whitespace dan newline di sekitar keyword

Ekspektasi: HARUS terdeteksi meski ada spasi ekstra

Selamat datang di situs gacor terpercaya.

Menangkan maxwin setiap malam.
